## Q1、OWASP top 10了解吗？说说

```
A01:2021-访问控制失效（Broken Access Control）：此类别从第五位上升至首位，表明94%的应用程序在某种形式上测试了访问控制失效问题。映射到访问控制失效的34个常见弱点枚举（CWEs）在应用程序中的出现次数比任何其他类别都多。

A02:2021-加密失败（Cryptographic Failures）：此类别从先前的敏感数据泄露（Sensitive Data Exposure）更名而来，上升一位至第二，现在聚焦于与加密相关的失败，这通常导致敏感数据泄露或系统妥协。

A03:2021-注入（Injection）：此类别下降至第三位。94%的应用程序测试了某种形式的注入，映射到此类别的33个CWEs在应用程序中有第二多的出现次数。现在此版中，跨站脚本攻击（XSS）也包含在此类别中。

A04:2021-不安全的设计（Insecure Design）：这是2021年的一个新类别，专注于与设计缺陷相关的风险。这要求行业“向左移动”，更多地使用威胁建模、安全设计模式和原则以及参考架构。

A05:2021-安全配置错误（Security Misconfiguration）：从上一版的第六位上升至此，90%的应用程序测试了某种形式的配置错误。随着软件变得高度可配置，看到此类别上升并不意外。

A06:2021-脆弱和过时的组件（Vulnerable and Outdated Components）：此类别以前称为使用已知漏洞的组件，从2017年的第九位上升，通过数据分析进入前十名，是一个我们难以测试和评估风险的已知问题。

A07:2021-身份识别和认证失败（Identification and Authentication Failures）：之前称为身份验证机制破坏，从第二位下降，并且现在包含了更多与身份识别失败相关的CWEs。

A08:2021-软件和数据完整性失败（Software and Data Integrity Failures）：2021年的一个新类别，关注在不验证完整性的情况下对软件更新、关键数据和CI/CD管道的假设。这个类别在CVE/CVSS数据映射到的10个CWEs中具有非常高的影响权重。

A09:2021-安全日志和监控失败（Security Logging and Monitoring Failures）：之前称为不足的日志记录与监控，从行业调查中加入，从先前的第十位上升。这个类别扩展到包括更多类型的失败，测试起来具有挑战性，并且在CVE/CVSS数据中代表性不强。

A10:2021-服务端请求伪造（Server-Side Request Forgery, SSRF）：从Top 10社区调查中加入，数据显示相对较低的发生率但测试覆盖率高于平均水平，以及对于利用和影响潜力的高于平均水平的评级。这个类别代表安全社区成员告诉我们这
```

## Q2、说说SQL注入，类型，报错注入的函数，防御措施有哪些？工具说说，常用命令？

```
类型：
原理性（Classic）SQL注入：通过插入或修改SQL命令，攻击者可以执行未授权的查询或数据操作。

盲注（Blind SQL Injection）：攻击者无法看到应用程序的响应数据，通过发送数据并观察应用的反应或行为来推断信息。

基于错误的SQL注入（Error-Based SQL Injection）：利用数据库错误信息来获取数据库的结构和内容。

联合查询注入（Union-Based SQL Injection）：通过使用UNION SQL操作符，攻击者可以通过一个查询语句获取数据库中的额外信息。

时间盲注（Time-Based Blind SQL Injection）：攻击者通过观察查询执行的时间来推断数据。

防御措施：
预编译语句与参数化查询：使用预编译语句和参数化查询是防止SQL注入的最有效手段之一，因为它们将SQL代码与数据分开处理。

使用ORM框架：对象关系映射（ORM）框架提供了一种更安全的方式来构建数据库查询，而无需手动编写SQL语句。

输入验证：验证所有输入数据，确保它们符合预期的格式，可以减少SQL注入的风险。

最小权限原则：确保数据库连接使用的账户仅具有执行当前操作所需的最小权限。

错误消息处理：不要向用户显示详细的数据库错误信息，以防泄露有助于攻击者的信息。

```
